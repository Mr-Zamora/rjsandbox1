How to Recreate This Website

This guide provides step-by-step instructions for recreating this Filipino Recipes website, focusing on modern web development best practices.

Core Web Technologies

  • HTML5 Semantic Structure: Use semantic tags like <header>, <nav>, <main>, <section>, <article>, and <footer> to create a meaningful document outline that improves accessibility and SEO.
  • CSS Layout Systems: Implement Flexbox for one-dimensional layouts and CSS Grid for two-dimensional layouts.
  • Responsive Design: Use media queries and relative units to ensure the site works on all device sizes.
  • Progressive Enhancement: Build the core functionality first, then add advanced features for modern browsers.

Pages and Component Structure

1. Index Page

Purpose: Display a responsive grid of recipe cards with clickable links to detailed recipe pages.

  • Header Component: Structured with <header class="header" role="banner"> containing a logo and navigation.
  • Recipe Grid Component: Uses <main class="content grid-4-col" role="main"> with <article class="card"> items rendered from data.

2. Recipe Detail Page

Purpose: Display detailed information about a specific recipe with a responsive two-column layout.

  • Layout: <main class="recipe-content" role="main"> containing the recipe image and detailed content.
  • Content: Includes title, description, ingredients, directions, and metadata like preparation time.

3. Add Recipe Page

Purpose: Provide an accessible form for submitting new recipes with validation feedback.

  • Form Structure: <form id="recipe-form" method="post"> with labelled inputs and ARIA hints.
  • Validation: Client-side validation combined with server-side handling when integrated with Flask.

4. About Page

Purpose: Describe the project, its technical specifications, and educational value.

  • Content: Explains the tech stack, design principles, and goals for students building the project.

5. CSS Organization

The CSS is organized into sections for variables, base styles, layout components, responsive adjustments, and accessibility helpers.

6. JavaScript Functionality

Enhances user experience with interactive features while maintaining accessibility, such as the mobile navigation toggle and planned form validation.

7. Transitioning to Flask

Move HTML files into the templates/ folder, static assets into static/, replace static links with {% raw %}{{ url_for('route_name') }}{% endraw %}, and prepare for database-backed content.
